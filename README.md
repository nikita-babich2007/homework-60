# Homework-67

## Курсори та Агрегації MongoDB

Проєкт оптимізовано для роботи з великими обсягами даних за допомогою нативних інструментів MongoDB Mongoose.

### 1. Потокове читання через Курсор

- **Маршрут:** `GET /articles/cursor`
- **Опис:** Використовує метод `.cursor()` для перебору документів бази даних по одному. Замість завантаження всієї колекції в оперативну пам'ять сервера (RAM), дані відправляються клієнту частинами (`res.write`). Це критично важливо для оптимізації при обробці тисяч записів.
- **Приклад відповіді:** Повертає стандартний JSON-масив зі статтями, але згенерований потоково.

### 2. Агрегаційний запит (Статистика)

- **Маршрут:** `GET /articles/stats`
- **Опис:** Використовує Aggregation Framework (`Article.aggregate()`) для перекладання складних обчислень на сторону бази даних.
- **Логіка пайплайну:**
  1. `$project`: вираховує довжину тексту (`title` та `content`) для кожного документа за допомогою `$strLenCP`.
  2. `$group`: об'єднує всі документи (`_id: null`) та вираховує загальну кількість статей (`$sum`), а також середню довжину заголовків і текстів (`$avg`).
- **Приклад очікуваної відповіді:**
  ```json
  {
    "_id": null,
    "totalArticles": 5,
    "avgTitleLength": 18.4,
    "avgContentLength": 125.2
  }
  ```
