# Homework-65

Цей проєкт — сервер на Node.js та Express.js за архітектурою MVC.

## 🍃 MongoDB Atlas (ДЗ 65)

- **База даних:** Сервер успішно підключено до хмарної бази даних MongoDB Atlas за допомогою бібліотеки `mongoose`.
- **Налаштування:** Рядок підключення (`MONGO_URI`) безпечно зберігається у файлі змінних оточення `.env`. Логіка підключення винесена в окремий файл `src/config/db.js`.
- **Моделі даних:** Створено Mongoose-схему та модель `Article` (`src/models/Article.js`) для роботи з колекцією статей.
- **Операції читання (Read):** \* Маршрут `GET /articles` зчитує всі записи з MongoDB за допомогою `Article.find()` та рендерить їх через EJS-шаблон.
  - Маршрут `GET /articles/:articleId` знаходить конкретний документ за допомогою `Article.findById()` та відображає деталі на сторінці.

## 🛡️ Passport.js та Сесії

- **Passport.js:** Інтегровано стратегію `passport-local` для авторизації з використанням `email` та `password`.
- **Маршрути:** Реалізовано ендпоінти `/auth/register`, `/auth/login` та `/auth/logout`.
- **Сесії:** Налаштовано використання `express-session` для тривалої авторизації. Ідентифікатор сесії зберігається в куках із прапорцем `httpOnly` (та `secure` для безпечних з'єднань).
- **Захист маршрутів:** Додано захищений маршрут `/protected`, доступ до якого контролюється за допомогою мідлвару та вбудованого методу `req.isAuthenticated()` від Passport.js.

## Cookies та JWT Авторизація

- **Статичні файли:** Налаштовано віддачу `favicon.ico` з папки `public`. Тег підключення додано у всі PUG та EJS шаблони.
- **Cookies (Налаштування):** Додано маршрут `POST /auth/theme`, який приймає тему користувача та зберігає її у cookies (використовується `cookie-parser`).
- **JWT (JSON Web Token):** \* Додано маршрути `POST /auth/register` та `POST /auth/login`. При успішній автентифікації генерується JWT.
  - Токен безпечно зберігається у cookies з прапорцем `httpOnly`.
  - Створено мідлвар `verifyJWT`, який захищає маршрути `/articles`, перевіряючи наявність та валідність токена з кук.

В проєкт інтегровано два шаблонізатори для серверного рендерингу сторінок (SSR):

- **PUG**: Використовується для маршрутів `/users` та `/users/:userId`. Шаблони генерують HTML зі списком користувачів та їхніми деталями.
- **EJS**: Використовується для маршрутів `/articles` та `/articles/:articleId`. Шаблони генерують HTML зі списком статей та їхнім контентом.
- Додано папку `public/css` з базовими стилями `style.css` для візуального оформлення сторінок.
