## 🛠 Рефакторинг на Mongoose (ДЗ 70)

### Що було зроблено:
1. **З'єднання:** Налаштовано стабільне підключення через `mongoose.connect()`.
2. **Схеми та Моделі:** Створено строгу схему даних (`Schema`), яка описує структуру документів у базі.
3. **Валідація:** Додано перевірки на рівні бази даних (наприклад, обов'язкові поля `required`, мінімальна довжина `minlength` для контенту). Тепер невалідні дані не зможуть потрапити до БД.
4. **Індексація:** Додано індекс (`index: true`) для поля `title`, що значно прискорить пошук за заголовками при збільшенні обсягу даних.
5. **Автоматизація:** Увімкнено опцію `timestamps`, яка автоматично генерує дати створення та оновлення документа.
6. **Контролери:** Усі запити до БД (CRUD операції) переписані з використанням сучасних методів Mongoose (`.find()`, `.save()`, `.findById()`).

### Як запустити

1. Переконайтеся, що у вас встановлено Docker та Docker Desktop.
2. Відкрийте термінал у корені проєкту та виконайте команду:
   ```bash
   docker-compose up --build
   ```
